Git Shell 
第一部分：
Mkdir 创建目录
Git init 初始化仓库
Git add <file> 添加文件
Git commit –m “”提交文件 –m 说明
Git config - - global user.email “”
Git config - - global user.name “”
Git status 查看仓库状态，如commit（提交）等
Git diff 查看改动文件内容的不同，具体到行。
Git log 查看最近的提交情况
Git log - - pretty=oneline 按一行显示log
Git reset - - hard HEAD^ 退回到上个版本 ^^退回到上两个版本 (HEAD~100退回到上100个版本)
Cat  file 查看完整文件内容。
Git reset - - hard 版本号   回到某一个版本，版本号不需要填所有位，前几位即可。
Git reflog 记录每一次命令




第二部分：
仓库同时也是工作区，.git隐藏文件是版本库，其中有个文件叫index(stage),是暂存区，还有git自动创建的第一个分支master，以及指向master的一个指针HEAD。
  Git add指令是把文件添加到暂存区，git commit是吧暂存区的文件提交到当前分支，所以git status命令检查的是工作区。
Git checkout - - file 把文件在工作区的修改撤销，一种是还没add的文件现在还原到版本库一样的状态，一种是已经add的文件又做了修改，恢复到添加到暂存区后的状态。
1.	当你改乱了工作区某个文件的内容，想直接丢弃工作区的修改时，用git checkout - -file
2.	当你不但改乱了工作区某个文件的内容，还添加到了暂存区时，想丢弃修改，分两步，第一步用 git reset HEAD file，回到了场景1，第二部按场景1操作。
3.	已经提交了不适合的修改到版本库时，想要撤销本次提交。
4.	rm file 删除工作区的文件。
5.	git rm file  git commit 删除版本库的文件
6.	git check out - - file 从版本库检出到工作区(如果版本库的文件被删除，则无法检出)。
